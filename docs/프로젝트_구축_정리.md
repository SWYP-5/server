## 각 파일 상세 설명

### 1. pom.xml — 라이브러리 관리 파일

Maven이라는 빌드 도구가 읽는 파일이에요.  
"이 프로젝트에 어떤 라이브러리가 필요한지" 목록을 정의합니다.

**추가한 주요 라이브러리:**

| 라이브러리 | 역할 |
|-----------|------|
| `spring-boot-starter-web` | HTTP API 서버 기능 |
| `spring-boot-starter-data-jpa` | Java 코드로 DB 조작 (SQL 직접 안 써도 됨) |
| `spring-boot-starter-security` | 로그인/권한 관리 |
| `spring-boot-starter-oauth2-client` | 카카오·구글 소셜 로그인 |
| `jjwt-api/impl/jackson` | JWT 토큰 생성·검증 |
| `mysql-connector-j` | Java ↔ MySQL 연결 드라이버 |
| `lombok` | 반복 코드(getter/setter 등) 자동 생성 |
| `spring-boot-starter-validation` | 입력값 유효성 검사 |

---

### 2. application.yml — 설정값 파일

서버가 시작될 때 읽어오는 설정 파일이에요.  
DB 접속 정보, 포트 번호, 소셜 로그인 키 등을 여기서 관리합니다.

**주요 설정 항목:**

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/app_db   # 어떤 DB에 접속할지
    username: root                             # DB 계정
    password: 1234                             # DB 비밀번호

  jpa:
    hibernate:
      ddl-auto: update   # Entity 클래스 보고 테이블 자동 생성/수정

jwt:
  secret: ...                    # 토큰 서명에 쓰는 비밀키
  access-token-expiration: 1800000   # 액세스 토큰 유효시간 (30분)
  refresh-token-expiration: 604800000 # 리프레시 토큰 유효시간 (7일)

server:
  port: 8080   # 서버가 열리는 포트 번호
```

> **중요:** 비밀키(JWT secret, 소셜 로그인 키)는 절대 Git에 올리면 안 돼요.  
> `.gitignore`에 `application-local.yml`을 추가해 두었으니,  
> 실제 키는 `application-local.yml` 파일에 따로 보관하세요.

### 3. AppBackendApplication.java — 서버 시작점

```java
@SpringBootApplication
public class AppBackendApplication {
    public static void main(String[] args) {
        SpringApplication.run(AppBackendApplication.class, args);
    }
}
```

- IntelliJ에서 ▶ 버튼을 누르면 이 `main()` 메서드가 실행돼요.
- `@SpringBootApplication` 하나로 자동 설정이 다 됩니다.

### 4. SecurityConfig.java — 보안 설정

Spring Security는 기본적으로 **모든 요청을 막아요.**  
이 파일에서 "어떤 주소는 로그인 없이 허용할지"를 설정합니다.

```java
.requestMatchers("/api/auth/**", "/api/health", "/login/**", "/oauth2/**").permitAll()
// ↑ 이 주소들은 누구나 접근 가능

.anyRequest().authenticated()
// ↑ 나머지는 로그인한 사람만 접근 가능
```

**현재 설정 상태:**
- 세션 없음 (JWT 토큰 방식이라 세션 불필요)
- CSRF 비활성화 (REST API는 CSRF 보호 불필요)
- JWT 필터는 팀원이 추가 예정

### 5. HealthController.java — 헬스체크 API

서버가 정상 동작하는지 확인하는 가장 간단한 API예요.

```
GET http://localhost:8080/api/health
```

응답:
```json
{
  "status": "UP",
  "message": "서버 정상 작동 중"
}
```

실무에서도 서버 상태 모니터링에 이런 헬스체크 API를 많이 써요.

### 6. .gitignore — Git 제외 파일

Git은 코드 변경사항을 추적하는데, 올리면 안 되는 파일이 있어요.

```
target/              ← 빌드 결과물 (자동 생성되므로 불필요)
.idea/               ← IntelliJ 개인 설정 (팀원마다 다름)
.env                 ← 비밀키/비밀번호 (절대 올리면 안 됨!)
application-local.yml ← 로컬 전용 설정 (역시 비밀 정보 포함)
```

## 패키지 구조 설명 (앞으로 쓸 폴더들)

```
controller/   ← 클라이언트의 요청을 받는 곳 (API 주소 정의)
      ↓
service/      ← 실제 비즈니스 로직 처리
      ↓
repository/   ← DB에 데이터를 저장/조회
      ↓
entity/       ← DB 테이블과 1:1 매핑되는 클래스
dto/          ← 요청(Request)/응답(Response) 데이터 형태 정의
exception/    ← 에러 처리
config/       ← 각종 설정 클래스
```

이 구조를 **레이어드 아키텍처**라고 해요. 각 계층이 역할을 분리해서 코드가 깔끔해져요.

## 자주 나오는 용어 정리

| 용어 | 설명 |
|------|------|
| **Spring Boot** | Java로 웹 서버를 빠르게 만들 수 있는 프레임워크 |
| **Maven / pom.xml** | 라이브러리를 자동으로 다운받아주는 도구 |
| **JPA** | Java 코드로 DB를 다루는 기술 (SQL 자동 생성) |
| **JWT** | 로그인 상태를 증명하는 토큰 (쿠키 없이 사용) |
| **OAuth2** | 카카오/구글 등 외부 서비스로 로그인하는 표준 방식 |
| **Spring Security** | 인증(로그인)·인가(권한) 처리를 담당하는 모듈 |
| **REST API** | 주소(URL)로 요청하고 JSON으로 응답하는 방식 |
| **포트 8080** | 서버가 열리는 문 번호. 브라우저에서 `localhost:8080`으로 접속 |
| **ddl-auto: update** | Entity 클래스 변경 시 DB 테이블 자동으로 맞춰줌 |
